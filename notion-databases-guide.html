01 — Fundamento

O que é um Banco de Dados

No Notion, um banco de dados é uma coleção de páginas com propriedades estruturadas — diferente de uma simples lista ou tabela, cada item é uma página completa com atributos tipados.

💡 Distinção fundamental: cada linha de um banco de dados Notion é, na verdade, uma página inteira com seu próprio conteúdo. As colunas (propriedades) são apenas metadados dessa página.

04 — Filtros

Filtros dinâmicos — o recurso mais subestimado

👤
Filtro "Eu"
Em propriedades do tipo Person, você pode filtrar por "Eu" — o Notion resolve isso para o usuário atual. Cada membro vê apenas seus itens.
📅
Datas relativas
Filtros como "prazo = próximos 7 dias" são sempre calculados a partir da data atual. A view se atualiza sozinha todo dia.
🔗
Filtro por Relation
Filtre entradas que possuem (ou não possuem) uma relação com determinado item de outro database.
⚠️ Limitação real: filtros no Notion são por view, não por database. Se você deletar uma view, perdeu os filtros configurados nela. E filtros com lógica mista (AND dentro de OR) só são possíveis criando grupos de filtro — recurso adicionado em 2023.

Tipos de relação

TipoDescriçãoQuando usar
Unidirecional Só o database A vê a relação. B não tem coluna automática. Quando B é um database auxiliar e você não quer poluir ele com referências.
Bidirecional Ambos os databases exibem a coluna de relação. Editar um lado reflete no outro. Quando você precisa navegar nos dois sentidos: do Projeto ver as Tarefas, e da Tarefa ver o Projeto.
Self-relation Um database se relaciona consigo mesmo. Hierarquias: Tarefa pai → subtarefas. Funciona com a propriedade nativa "Sub-items".
ℹ️ Sub-items vs Relation manual: desde 2023 o Notion tem sub-items nativos (hierarquia dentro do mesmo database). Para hierarquias simples, isso é mais fácil que criar uma self-relation manualmente.
06 — Rollups

Rollups — Agregações entre Databases

Rollup é uma propriedade que atravessa uma Relation e agrega valores do database relacionado. Sem Rollup, você conecta projetos a tarefas, mas não sabe quantas tarefas tem cada projeto. Com Rollup, você sabe — e pode calcular soma, média, contagem, e muito mais.

🧩 Pré-requisito: para criar um Rollup você precisa de uma Relation já configurada. Rollup sempre depende de uma Relation existente — ele pergunta: "de qual relação? qual propriedade? qual cálculo?"

💡 Rollup de Rollup: é possível criar um Rollup que agrega o resultado de outro Rollup — útil para hierarquias de 3 níveis (ex: Portfolio → Projetos → Tarefas). Mas cuidado: a performance pode degradar com muitos itens e camadas.
⚠️ Limitação real: Rollups não reagem a filtros da view em tempo real. Eles agregam todos os itens relacionados, independente do que está visível na view. Para cálculos condicionais, você precisa combinar Rollup + Fórmula.
07 — Fórmulas

Fórmulas — lógica dentro do database

A propriedade Formula permite cálculos dinâmicos baseados em outras propriedades da mesma entrada. Com a Fórmula 2.0 (lançada em 2023), o poder aumentou significativamente — variáveis, condicionais complexas, e até acesso a itens de Relation.

Exemplos práticos

// Calcular dias restantes até o prazo
dateBetween(prop("Prazo"), now(), "days")
// Status legível com emoji
if(prop("Concluído"), "✅ Feito",
  if(prop("Status") == "Em andamento", "🔄 Em progresso",
    "⏳ Pendente"))
// Progresso % combinando Rollup e Number
round(prop("Tarefas Concluídas") / prop("Total Tarefas") * 100) + "%"
// Fórmula 2.0 — usando variáveis (let)
let(diasRestantes, dateBetween(prop("Prazo"), now(), "days"),
  if(diasRestantes < 0, "🔴 Atrasado",
    if(diasRestantes <= 3, "🟠 Urgente",
      "🟢 No prazo")))
CategoriaFunções principais

Númeroround, floor, ceil, abs, max, min, mod, sqrt
Datanow, dateAdd, dateBetween, formatDate, day, month, year

08 — Ordenação

Ordenação (Sort)

Ordenações definem a sequência dos itens na view. Assim como filtros, cada view tem suas próprias ordenações independentes. Você pode empilhar múltiplas ordenações — a primeira tem prioridade.

Agrupamento (Group by)
